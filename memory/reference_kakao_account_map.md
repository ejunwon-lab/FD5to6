증권사 카톡 매매 체결 알림을 *거래_원장*에 기록할 때 쓰는 매핑. 카톡엔 마스킹 계좌번호만 오므로 실계좌·수수료를 여기서 조회. 새 마스킹 계좌 등장하면 사용자에게 묻고 누적.

기록 경로: `action=addTrade` doPost → `_appendTradeRow`(NewSystem.js). 호출 헬퍼 `scripts/post_trade.py`(URL 자동·secret=.env). 설계: `docs/plans/2026-06-11-카톡매매-원장자동기록.md`.

## 계좌 매핑 (마스킹번호 → 증권사 / 계좌 / 수수료규칙)
| 마스킹 계좌번호 | 증권사 | 시스템 계좌 | 수수료 |
|---|---|---|---|
| `037-99-****97` | 미래에셋투자증권 | 종합_랩 | **없음** (무수수료) |
| `71528*****-14` | 삼성증권 | ISA | **국내ETF 매도 ≈ 0.0042%** (원장 실측: 화장품 740/[금액]·원자력 545/[금액]·로봇 1008/[금액] 모두 0.0042%). 국내주식은 거래세 포함 별도(~0.15%). 카톡엔 수수료 미포함 → 추정 기록 |

## 증권사명 정규화
- 카톡 `미래에셋증권` → 시스템 상수 `미래에셋투자증권`
- 카톡 `삼성…` → `삼성증권`

## 삼성증권 카톡 포맷 (2026-07-14 확보)
```
[삼성증권]<주식체결안내>
71528*****-14          ← 마스킹 계좌
TIGER 반도체TOP10       ← 종목명만 (코드 없음 → 보유현황에서 코드 룩업)
매도11주 [금액]원       ← 구분+수량+단가 (수수료 없음)
체결(1차)              ← 부분체결 차수 (같은 종목 여러 차수 → 수량 합산해 1행 기록)
```
- 코드 없음 → *보유현황*(종목명·증권사·계좌)에서 코드 조회. `backup_sheets.py`로 시트 덤프 후 grep이 확실.
- 같은 종목 여러 차수(1차·2차…) → **수량 합산**해 1행으로 기록(단가 동일).

## 파싱 규칙
- 종목코드: `실리콘투(A257720)` → 앞 `A` 제거 → `257720` (GAS `_normCode`는 7자 A접두를 안 자름).
- 금액: 카톡 체결금액 = 수량×단가(gross, 수수료 미포함) — 검산용.
- 날짜: 카톡에 없음 → 사용자가 안 주면 **받은 날(오늘 KST)**.
- 분류: 카톡에 없음 → 매도는 GAS가 *보유현황*(코드+증권사+계좌)에서 룩업, 신규매수는 사용자에게 질문.
