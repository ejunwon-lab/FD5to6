1
📱

💌

3

↓
4
🤖

5

🛡️ 4가지 안전장치

🔒 비밀번호 두 개

편지에는 도장(비밀번호)이 있어야 받음. 클라우드가 한 번 검사, 구글 봇이 한 번 더 검사. 두 번 통과해야 처리됨. 도둑은 못 들어옴.

📋 명단 (화이트리스트)

봇은 명단에 적힌 사람한테만 답장함. 처음 보는 사람은 그냥 무시. 단, "/whoami" 명령은 모르는 사람도 OK — 자기 ID를 알아내는 데만 쓰여서 안전.

⏸️ 한 번에 하나만

"ㄱㄱ ㄱㄱ ㄱㄱ" 빠르게 여러 번 보내면 첫 번째만 일하고, 나머지는 "잠시만요" 답장. 봇이 헷갈리지 않게 줄을 세움.

🔁 같은 편지 두 번 받으면 무시

네트워크가 이상해서 같은 메시지가 두 번 와도, 두 번째는 슬쩍 버림. 손익 답장이 중복으로 안 옴.

🌧️ 문제가 생긴다면?

☁️ 클라우드가 잠시 다운

편지 전달 못 함. 다시 살아나면 자동으로 처리. 평소엔 거의 안 일어남 (99.99% 작동).

🤖 구글 봇이 일하는 데 시간 좀 걸림

가끔 시트 갱신이 오래 걸리면 텔레그램이 "답이 늦네?" 하고 다시 편지 보냄. 봇은 "어, 이거 이미 받은 거네!" 하고 무시. 중복 답장 안 옴.

✅ 평소엔?

아무 문제 없음. 모두 잘 동작하고 있음.

🧠 어른들이 알아두면 좋은 것

⚙️ 코드 수정 시 새 deployment 만들기 필수

handleTelegramWebhook 함수를 고치면 GAS의 "배포 → 배포 관리 → 새 버전" 한 번 실행해야 적용됨. 안 하면 옛 코드가 계속 돌아감.

☁️ Cloudflare env 변경 시 Deploy 수동 클릭

SECRET·GAS_URL 같은 환경변수 바꾸면 Cloudflare 대시보드 "Deployments" 탭에서 최신 version을 "Deploy this version" 눌러야 반영됨.

🔗 GAS deployment URL이 바뀌면

새 deployment를 만들었거나 URL이 변경되면 Cloudflare Worker의 GAS_URL env 값도 같이 갱신해야 함.
