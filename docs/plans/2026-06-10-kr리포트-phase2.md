## 2a. GAS 지표 (익스포저%·MDD)
### 1. 외부 동작 가정 + 근거
- 보유현황 `분류`=자산군(국내주식/해외ETF/펀드…), **테마(반도체) 아님** → 근거: NS.CATEGORIES. 테마% = 보유 리스트 기반 매핑 필요.
- 추이기록 일별 추이(N~S col, row2 최신 upsert)에 일별 총자산 시계열 존재 → MDD 계산 가능. 근거: Trend.js 헤더 주석.
- GitHub 러너 → GAS 웹앱 fetch 가능 → 근거: telegram-push 입증(ANYONE_ANONYMOUS).
- doPost는 현재 `addMarketReport`·`pushPnL`만 분기 → `portfolioMetrics` 추가 필요. 근거: Main.js:86,89.
### 설계
- 신규 GAS `getPortfolioMetrics()`: 보유현황·추이기록 읽어 **상대 지표만** 반환 — `{ assetClassWeights:{국내주식:%,…}, holdings:[{name,category,weight%}], mdd:{pct, peakDate, curDate} }`. **원화 절대액 절대 미포함**(public 리포트 노출 방지).
- doPost에 `action==='portfolioMetrics'` 분기 → secret 검증 → JSON. (읽기 전용, 시트 변경 0)
- 테마 익스포저(반도체%): **에이전트가** holdings weight% + 테마 지식(보유 리스트)으로 합산. GAS는 테마 미관여(단순 유지).
- 프롬프트: 리스크 레이더에서 GAS metrics WebFetch → 익스포저%·MDD 표기. 실패 시 Phase1 정성 집중도로 degrade.
### 검증
- GAS: push_safe + 버전배포 → `node --check` + walk-through(MDD: 추이기록 [100,120,90,110] → peak120, trough90 → MDD -25%). 
- dry_run으로 러너가 GAS metrics fetch→리포트 반영 확인. **원화 절대액 안 나오는지 본문 확인**(privacy 게이트).

## 2b. 기회 자동선별 (섹터 구성종목)
### 1. 외부 동작 가정 + 근거
- Naver 섹터 구성종목 = `finance.naver.com/sise/sise_group_detail.naver?type=upjong&no=XXX` (no=업종코드, 업종 페이지에서 매핑) → 구성종목 표 → `[검증 필요·실환경]`.
- 에이전트 WebFetch로 이 페이지 파싱 가능? → **러너 진단 전 단정 금지**.
### 2. 과거 부류 → 회피책
- **수급/업종 N/A "Naver 차단은 LLM 추측"·routine 403**(errors.md): 외부 엔드포인트 작동 추론 단정 트랩. → **러너 진단(`_kr_diag` 워크플로로 실제 curl) 후에만 하드 의존.** 못 뽑으면 지식 기반 대표주〔미검증〕로 degrade.
- **git push|tail 트랩**(errors.md 2026-06-10): 배포·반영 주장은 origin 직접 grep 검증.
### 3. 추론 vs 실환경
- 머리로: GAS 함수 로직·MDD 계산·테마 합산·degrade 설계.
- 실환경 전용(진단으로): Naver sise_group_detail 러너 fetch/파싱, GAS metrics 러너 fetch, 업종→no 매핑 안정성.
### 4. 검증
- 2b: `_kr_diag.yml`(throwaway)로 러너에서 sise_group_detail 후보 URL curl → 구성종목 파싱 가능성 확인 후 프롬프트에 하드 반영. 검증 후 워크플로 제거.

→ **게이트**: 2a는 GAS walk-through+dry_run(privacy 본문 확인) 통과 시 진행. 2b는 **러너 진단 통과 후에만** 자동선별 하드 의존, 실패 시 degrade 유지.
