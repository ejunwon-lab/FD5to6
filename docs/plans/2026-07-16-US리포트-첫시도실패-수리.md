# 설계 노트 — US 리포트 첫 시도 실패 수리 (permission bypass + 턴 증가 + 파일명 결정론화)

2026-07-16. `/design-check` 산출물. CI/스케줄 파이프라인(market-report.yml) 변경 = 위험 변경.

## 증상·원인 (실측)

- **실패 5회/9일** (7/6×2·7/12·7/13·7/14, 전부 US 창 23:0xZ 첫 dispatch). 체인 재시도(23:2x)가 매번 성공해 **리포트 자체는 매일 도착** — 사용자 체감은 아침 ⚠️ 알림+빨간 run.
- 실패 run 로그(29375054682): 생성 step exit 0인데 파일 미생성 → 에이전트 최종 발화가 **"파일 쓰기 권한 승인이 필요합니다"** 산문 + 파일명도 **US-2026-07-14**(US 마감일)로 오계산(기대 US-2026-07-15=KST). 실패 run 16분 vs 성공 run 9분 — 아침 Yahoo 429 재시도(errors.md 2026-07-03)로 턴 소모 후 이탈하는 패턴.
- `--allowedTools`에 Write 포함돼 있고 성공일은 같은 조건에서 정상 저장 → 결정적 재현 불가한 **에이전트 비결정성**(권한 산문 이탈 + 날짜 자가계산 오류) 이 원인 부류. 파이프라인이 이 비결정성에 무방비.

## 변경 3건 (us·kr·weekly 3개 생성 step 일괄)

1. `--permission-mode bypassPermissions` 추가 — "권한 승인 대기" 실패 부류 원천 제거. 보안 등가: 이미 Bash 무제한 허용이라 권한 표면 증가 없음(러너는 일회용, non-root `runner` 유저).
2. `--max-turns 25 → 40` (us·kr; weekly는 이미 40) — 429 재시도 아침의 턴 소진 여유.
3. **파일명 결정론화**: 생성 step에서 `DATE_KST=$(TZ='Asia/Seoul' date +%Y-%m-%d)` 계산 → 프롬프트 끝에 `[운영 지시] 저장 파일명은 반드시 docs/reports/{US|KR|WEEK}-${DATE_KST}.md — 날짜를 스스로 계산하지 말 것` 주입. path 판정 step과 동일 식이라 어긋남 구조적 불가.

알림 step·가드·체인 재시도는 불변 (안전망 유지).

## 1. 외부 동작 가정 + 근거

- CLI가 `--permission-mode bypassPermissions` 지원 → **[사실]** 로컬 `claude --help` 실측 (mode 목록에 명시).
- bypassPermissions는 non-root 러너에서 추가 플래그 없이 동작 → **[사실상 확실 — dry_run 스모크로 실측]**.
- path 판정 식 = `TZ='Asia/Seoul' date +%Y-%m-%d` (US·KR·WEEK 동일) → **[사실]** 워크플로 grep.
- `workflow_dispatch` `dry_run=true`는 생성만 하고 발송·commit·알림 skip → **[사실]** 각 step if 조건에 `inputs.dry_run != true` 확인.
- 프롬프트는 private repo `_config/`에서 cat — 주입 라인은 워크플로 쪽이라 public에 종목정보 노출 없음 → **[사실]**.

## 2. 과거 부류 (errors.md)

- **2026-07-03 Yahoo 429·WebFetch 차단** — 턴 소모의 근원. curl 전환은 이미 반영돼 있고, 이번엔 턴 예산을 weekly와 동일(40)로.
- **2026-07-03 무음 실패 알림** — 알림 step 불변. 이번 수리로 알림 발생 빈도 자체를 줄이는 것.
- **2026-06-10 daisy-chain** — 재시도 안전망이 있어 지금도 최종 도착은 보장. 이번 변경은 첫 시도 성공률 개선.

## 3. 추론 가능 vs 실환경 전용

- 머리로 거름: 플래그 의미, 보안 등가성(Bash 이미 무제한), 파일명 식 일치, dry_run 경로.
- 실환경에서만: ⓐ bypassPermissions가 러너에서 실제 수용되는지 ⓑ 주입 라인을 에이전트가 준수하는지 ⓒ 내일 08:05 첫 시도 성공 → **dry_run 스모크 + 내일 라이브 run으로만 판정**.
