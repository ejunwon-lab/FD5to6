# 설계 노트 — 자동화 헬스체크 watchdog (2026-07-19)

목적: **"아무것도 실행되지 않은" 침묵 실패**를 매일 저녁 1회 대사해 텔레그램으로 가시화.
기존 알림(체인 끊김 alert·리포트 생성실패 alert)은 전부 "실행됐지만 실패"만 잡는다 —
실행 자체가 없으면(체인 사망+cron 드롭, 트리거 부재) 아무도 모른다. 그 구멍이 6/5·6/9·7/9의 공통 뿌리.

## 구조 결정

- **`watchdog.yml`** (신규, cron 21:10·22:10 KST + workflow_dispatch) → **`scripts/watchdog_check.sh`** (검사 로직, 로컬 실행 가능) → 텔레그램 발송.
- **daisy-chain은 건드리지 않는다** — 크리티컬 경로(telegram-push.yml) 무수정. watchdog은 cron만 쓴다.
  저빈도 cron은 6/9~10 실측에서 "지연은 있어도 발화는 됨"(market-report 4/4 발화) — 지연 허용 가능한 일일 점검엔 충분.
- **매일 heartbeat 발송** (이상 없어도 ✅ 1줄) — "경보 없음 = 정상 or watchdog 사망" 모호성 제거.
  watchdog 자신의 사망은 사용자가 "저녁 메시지 부재"로 감지 (감시자의 감시자 문제를 사람 습관으로 해소).
- 21:10·22:10 이중 cron의 중복 발송 방지: 오늘 KST 성공 run이 이미 있으면 skip (`gh run list` 자가 조회).
- 검사 실패(네트워크 등)는 ⚠️ 확인불가로 보고 — watchdog 자체는 죽지 않게 항목별 격리.

## 1. 외부 동작 가정 + 근거

- GAS 웹앱 POST는 `-X POST` 금지, `--data`+`-L` → errors.md 2026-06-06 (기존 pre-fetch step과 동일 패턴 복사).
- `portfolioMetrics` 응답에 `dailyReturns:[{date,...}]` 존재 → MobileAPI.js:340·367 확인. 거래일 행만 포함(`_isTradingDateStr` 필터) → 거래일 저녁엔 last date == 오늘.
- 푸시는 `tgPushPnL`→`updateAllNew`→`logToTrendSheet` 경로라 거래일엔 추이기록에 오늘 행이 반드시 생김 → errors.md 2026-06-10 (v11 전체 갱신 전환).
- telegram-push run 로그에 `resp={"success":true,"result":"sent|skip-holiday|…"}` 에코 → telegram-push.yml:107, Telegram.js:153 (v13).
- 21:10 KST엔 장중(09~16시) poke를 담은 run이 전부 completed → 체인 run 최대 120분이라 16시 poke 포함 run은 늦어도 ~18시 종료.
- `GITHUB_TOKEN`으로 자기 repo run 목록·로그 조회 가능 (`actions: read`) → 기존 telegram-push가 `actions: write` 사용 중인 같은 토큰.
- private 리포트 repo는 `REPORTS_DEPLOY_KEY`로 checkout → market-report.yml 3개 job 동일 패턴.
- 신규 secret 불필요 — TG_BOT_TOKEN·TG_CHAT_IDS·GAS_WEB_APP_URL·TG_WEBHOOK_SECRET·REPORTS_DEPLOY_KEY 전부 기존.

## 2. 과거 부류 (errors.md)

- **6/5 GAS 트리거 누락·6/9 cron 21중2 발화** — watchdog 자신도 best-effort cron 위 → (a) 저빈도 2회 cron(고빈도일수록 드롭 패턴의 역), (b) heartbeat 부재를 사람이 감지, 로 이중 완화. 정시성 불요(지연 발화도 검사 가치 동일).
- **6/10 `push | tail` exit code 은폐** — 파이프 뒤 exit code 판정 금지. 스크립트는 `set -u` + 명시적 캡처.
- **6/6 curl -X POST 405** — 회피 (위 가정).
- **7/17 제헌절 (달력 낡음)** — watchdog은 휴장 판정을 GAS 에코에서 *관측*만 하고 자체 달력을 두지 않는다 → 화이트리스트 중복 생성 안 함. 한계: "GAS 달력 자체가 틀린" 케이스는 미검출 (독립 KRX 소스 도입 = 별도 개선안 #1).
- **6/8 안전망 2회 레이스** — watchdog은 read-only + 자가 dedup(성공 run 조회)이라 중복 실행 무해(최악 메시지 2통).

## 3. 추론 가능 vs 실환경 전용

- 머리로 거름: 검사 로직·jq 파싱·KST 날짜 경계(`date -u -d "${KST일자} 00:00 +0900"`)·dedup·격리.
- 실환경에서만: ① gh run 로그 grep이 실제 sent 카운트를 내는가 ② GAS portfolioMetrics 실응답 스키마 ③ Actions 러너에서 secrets 주입·텔레그램 도달 → ①②는 **로컬에서 실측 가능**(gh 인증됨·.env 보유 — 오늘 즉시 스모크), ③은 push 후 workflow_dispatch 1회 라이브 스모크.

## 4. 검증 방법

- 자동: `bash -n`·yaml 파싱 → 통과. 로컬 dry-run(`scripts/watchdog_check.sh` 실발송 없이) → 오늘(토, 비거래일) 기준 "체인 alive·sent 0·휴장/주말·리포트 없음(토 정상)" 일관 출력.
- 라이브: push 후 `gh workflow run watchdog.yml` → run success + 텔레그램 heartbeat 1통 실도착.
- 사용자: **텔레그램 채널**에서 매일 **21:1x~22:1x KST** 사이 `🩺 자동화 헬스체크` 메시지 1통 → 거래일엔 "발송 N건(N≥1)·시트 오늘·리포트 US/KR ✓" 전부 ✅.

→ 게이트: **통과** (실환경 항목은 로컬 스모크 + 라이브 dispatch 1회로 당일 해소)
