# 설계 노트 — TWR 입금 왜곡 보정 (①원장 입출금 행 + ②이상치 플래그)

## 문제 재확인

- 총자산 dRate(추이기록 S열)가 입출금을 수익으로 오인 — 6/28 주간 리포트 d5 +4.9% vs 실질 약 -6%.
- **라이브 재현 (오늘)**: 7/16 일별추이 S = **-4.42%**인데 당일 손익은 ~0원. 전일 대비 총자산이 손익 없이 크게 감소(입출금/대기자금 수동 조정 추정) → 이번 일요일 주간 리포트가 또 왜곡될 상황.
- 입출금이 일자별로 기록되는 곳이 없음(원장은 매수/매도만, 대기자금은 *설정* 시트 수동 셀 덮어쓰기 = 상태, 이력 아님).

## 설계 요지

**스키마 원칙(전역 CLAUDE.md): 상태가 아닌 이력(append)으로.** → *거래_원장*에 `구분=입금/출금` 행 타입 추가 (이력, 소급 기록 가능).

1. **① 기록**: 원장에 입금/출금 행. 컬럼 규약 — `날짜·구분(입금/출금)·종목코드 빈칸·종목명 빈칸·분류 빈칸·증권사·계좌·수량 빈칸·단가 빈칸·금액(₩)·수수료 0·메모`. 기록 경로 2가지: ⓐ 시트 직접 행 입력 ⓑ `addTrade` POST 확장(Claude에게 "입금 기록해줘" → `post_trade.py`).
2. **② 보정+플래그 (read-time)**: `getPortfolioMetrics`(MobileAPI.js)가 원장에서 flow 맵(날짜→순입금)을 만들어 dailyReturns/d5/d20을 **읽기 시점에 보정**. S열 원본은 불변(Trend.js 무수정 — 이력 원본 보존, 소급 기록 시 자동 자가치유).
   - 보정식(end-of-day flow 관행): `V_prev = Q_t / (1 + S_t/100)` (Q=총자산 col17, S=dRate col19) → `r_adj = (Q_t − F_t − V_prev) / V_prev × 100`. F=0이면 r_adj ≡ S_t (기존과 완전 동일).
   - 비거래일 입금은 다음 거래일로 롤포워드(`_isTradingDateStr` 재사용) — 월요일 행 diff에 주말 입금이 들어있기 때문.
   - **플래그**: |dRatePct| ≥ 3.0 && 해당일 flow 기록 없음 → `suspect: true` (입출금 미기록 의심). 보정된 날은 `flowAdj: true` + `rawPct` 동반.
   - **원화 절대액 미노출 유지**: 응답에 flow 금액(₩)은 절대 포함 안 함 — boolean·%만. (public 리포트 정책)
3. **프롬프트**: `docs/reports/_config/weekly-prompt.md`에 flowAdj/suspect 의미 1단락 추가(suspect 날은 "입출금 미기록 가능성" 문구로 다루기).
