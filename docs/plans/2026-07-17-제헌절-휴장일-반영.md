## 원인 체인

1. 2026-05-11 발효된 법 개정으로 제헌절(7/17)이 18년 만에 공휴일 재지정 — 첫 적용일이 오늘. KRX 전 시장 휴장 (뉴스 다수 교차 확인).
2. *휴장일* 시트의 마지막 동기화는 2025-12월 `scheduledHolidaySync` — **재지정 발효 이전**이라 2026-07-17 행 없음.
3. 지금 `syncHolidays`를 다시 돌려도 `HOLIDAY_NAMES` 화이트리스트에 '제헌절'이 없어 걸러짐 — 스승의날 사고(errors.md 2026-05-17)의 해결책이었던 화이트리스트가 **법 개정으로 낡음** (화이트리스트의 구조적 약점: 세상이 바뀌면 코드도 바뀌어야 함).
4. `tgPushPnL` → `_mIsTradingDay` → `_isKoreanHoliday` → 시트 runtime 읽기 → 오늘=거래일 오판 → 발송.

## 변경 내용 (Holidays.js 3곳)

3. `_HOLIDAY_FALLBACK`에 `'2026-07-17'` 추가 — 시트 비었을 때 안전망 정합.

## 3. 추론 가능 vs 실환경 전용

- **머리로 거름**: 판정 체인(위 1), syncHolidays의 '수동' 행 보존 로직, 동기화 범위가 올해+내년이라 2025 행 드롭됨(기존 12월 동기화와 동일한 설계 동작 — 과거일 look-back은 최근 1주~1달이라 영향 없음), 고정날짜가 주말과 겹치는 미래 연도에도 무해(주말은 getDay로 먼저 걸러짐).
