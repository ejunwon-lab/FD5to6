> 2026-07-15. 위험 변경(배포 서버코드 `updatePositionFromLedger` — 모든 갱신 경로 통과 + doPost addTrade/pushPnL 경로) → `/design-check`.

## 문제 (검증 완료)
*보유현황* 우리은행 정기예금 5년 행: **평가금액(K) 빈칸 + 손익(L)=[금액](=전체 운용수익 집계 수식)**.
- opProfit = Σ평가 − Σ매입에서 예금 평가=0(빈칸)·매입 [금액] 포함 → **운용수익 = [금액] − [금액] = [금액]** (정상 [금액]보다 [금액] 적음). 산술 정확 일치로 원인 확정.
- 예금은 `NS.KIS_SKIP`이라 [updatePositionFromLedger:685-694·720-730](../../apps-script-v2/NewSystem.js) 이 값·수식을 **보존만** 함 → 빈칸·잘못된 수식이 영구 지속.
- 정상 대조: 펀드·보험 4행은 평가금액이 진짜값(현재단가와 다름, 예: 미래에셋차이나 [금액]≠[금액])이고 손익=평가−매입 성립 → 건드리면 안 됨.

## 수정 규칙 (최소 변경, 펀드·보험 무손상)
KIS_SKIP 행 처리 시:
- **평가금액(10) 있으면(>0) 그대로** (펀드·보험 = 진짜값 보존).
- **평가금액 비었고 현재단가(9)>0 → 평가금액 = 현재단가×수량** (예금 자가치유). 이 행을 `skipAutoFilled`로 표시.
- 자가치유 행만 **손익=평가−매입·수익률=손익/매입 재계산(값)** + **수식복원에서 평가/손익/수익률(인덱스 10·11·12) 제외** → 잘못된 집계 수식 제거.
- 펀드·보험(평가 present)은 auto 아님 → 기존 그대로(값·수식 복원) = **완전 불변**.
