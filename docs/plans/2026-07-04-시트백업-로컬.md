# 설계 노트 — 시트 백업: GAS backupData action + 로컬 자동 다운로드

2026-07-04. 사용자 결정: repo 적재 ✗, Drive 사본보다 **로컬 다운로드** 선호. 프로젝트 폴더 내 전용 폴더.

## 1. 외부 동작 가정
- GAS doPost 신규 action `backupData`(secret 게이트) — 기존 `portfolioMetrics`/`addTrade`와 동일 보안 모델(TG_WEBHOOK_SECRET POST) → 검증된 패턴(사실).
- 응답에 **원화 절대액 포함**(원장·보유현황 전체) — 공개 채널 아님(시크릿 게이트 + 로컬 저장 + `backups/` gitignore)이라 원화 금지 규칙 비적용. **repo에 절대 커밋 안 됨**이 게이트.
- 전 시트 `getDataRange().getValues()` JSON 직렬화 — Date는 ISO 문자열로 자동 직렬화(사실). ContentService 응답 상한(수십 MB)에 시트 전체(수천 행 규모)는 여유 → [검증: 실행 1회로 크기 확인].
- 로컬 인증: `scripts/post_trade.py`의 clasp 토큰→고정 웹앱 URL 발견 + `.env` 시크릿 패턴 재사용(사실 — 카톡 기록에서 검증됨).
- launchd(macOS)가 주 1회 스크립트 실행 — 표준 LaunchAgent 동작. 잠자기 중이면 다음 깨어날 때 실행됨(StartCalendarInterval 특성).

## 2. 과거 부류 (errors.md)
- GAS curl 진단 금지(memory) — 검증 시 python(requests)로 확인(post_trade 패턴이 이미 그러함).
- 무단서 실패 부류 — 백업 실패도 무음이면 안 됨 → 스크립트가 실패 시 exit 1 + 로그 파일 기록, launchd 로그로 확인 가능. (텔레그램 경고까지는 과설계로 판단 — 로컬 개인 백업.)

## 3. 추론 가능 vs 실환경 전용
- 머리로 거름: doPost 분기 추가 안전성(early return), secret 게이트, gitignore로 커밋 차단.
- 실환경에서만: 응답 크기·직렬화 실물, launchd 첫 발화 → **구현 직후 수동 1회 실행**으로 CSV 실물 검증, launchd는 `launchctl kickstart`로 즉시 1회 강제 실행해 확인.

## 4. 검증
- 자동: push_safe node --check → v26 재배포 → `python3 scripts/backup_sheets.py` 실행 → `backups/<날짜>/` 안에 시트별 CSV + 행수 출력 → 원장 행수가 시트 실제와 일치(대략 110+) → `git status`에 backups 안 뜸(gitignore).
