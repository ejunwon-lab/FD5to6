- 커밋 step `git add docs/reports/` → 에이전트가 거기 쓴 파일(파이프라인 상태) **함께 커밋**(다음날 Read), 별도 push 없음.

## 1. 외부 동작 가정 + 근거
- 에이전트가 `kr-theses.md`·`_kr_pipeline.md` Read, 파이프라인 상태 Write → 근거: allowedTools Read/Write + checkout. **✓확정**
- 다일 섹터 추세·후보 누적 = 전날 `_kr_pipeline.md` 이어읽기로 구현(외부 API 0) → **✓확정**
- 정성 집중도(반도체 9/16종) = 보유 리스트로 계산 → **✓확정**
- 로테이션 OUT/IN = 오늘 업종 데이터로 도출 → 현재 업종 작동(6/10 정상). 단 6/8 N/A 전력 → degrade 유지. 근거: errors.md.
- 섹터 구성종목 자동선별 엔드포인트(Naver sise_group_detail 등) 러너 작동 → `[검증 필요·실환경]` → **Phase 2**
- GAS portfolio metrics(익스포저%·MDD) 러너 fetch → 연결 입증(GitHub→GAS, telegram-push), 단 **새 GAS 함수+배포 필요** → **Phase 2**

## 2. 과거 부류 (errors.md) → 회피책
- **수급/업종 N/A·"Naver 차단은 LLM 추측"·routine 403 "Host not in allowlist"**(errors.md): 외부 엔드포인트 작동을 추론으로 단정한 트랩. → **회피**: 새 외부 엔드포인트(섹터구성·GAS metrics)는 **러너 dry_run/진단 검증 후에만 하드 의존**. Phase 1은 외부 신규 의존 0. 못 뽑는 칸은 N/A·degrade(리포트 전체 실패 금지 — 기존 프롬프트 규칙 유지).
- **KR push rejected race**(errors.md 2026-06-08): 파이프라인 상태 파일을 **리포트와 같은 commit step**에서 함께 커밋(별도 push 없음). ✓
- **"미국 IP 프로브 200 무효"**(다른 환경 검증): 검증은 **GitHub 러너 dry_run으로만**, 로컬 curl 신뢰 금지.

## 3. 추론 가능 vs 실환경 전용
- **머리로 거름 (Phase 1, 지금 구현)**: 파일 기반 파이프라인 상태·thesis·다일추세(prior)·정성 집중도·로테이션(오늘 업종)·degrade 설계. 기존 도구 권한 내, 외부 신규 의존 0.
- **실환경 전용 (Phase 2, dry_run/진단 후)**: 섹터 구성종목 엔드포인트 작동·GAS metrics fetch·업종 엔드포인트 간헐 실패율.

## 4. 검증 방법
- **자동**: Phase 1 파일 작성 후 **dry_run** → 로그 "REPORT 본문"에서 ①로테이션 OUT/IN ②스코어보드+thesis+수급 ③정성 집중도 ④파이프라인 ①②③ ⑤종합+관찰트리거 + `_kr_pipeline.md` 갱신 확인. 발송·커밋 없음.
- **Phase 2 전**: 섹터구성·GAS metrics를 dry_run/진단 워크플로로 러너 작동 확인 후 하드 의존.
- **사용자**: 다음 거래일 실제 리포트 형식 확인 + `kr-theses.md` 논리 초안 검토/수정.

→ **게이트: Phase 1 통과** (외부 신규 의존 0, dry_run 검증 가능). **Phase 2**(섹터 구성종목 자동선별·GAS 익스포저%/MDD)는 러너 검증 후 별도 진행.
