# 설계 노트 — docs/reports를 private repo(FD5to6-reports)로 분리

2026-07-04. 배경: FD5to6는 PUBLIC(GH Pages 의존) — `docs/reports/`에 보유 종목·비중%·MDD·투자 논리가 매일 커밋되어 공개됨. 사용자 결정: 리포트만 별도 private repo.

## 구조 결정
- **private repo `ejunwon-lab/FD5to6-reports`를 워크플로에서 `docs/reports/` 경로에 checkout** — 프롬프트·가드·weekly 읽기·`_kr_pipeline.md` 등 기존 경로 참조 전부 무수정. 커밋 step만 그 하위 repo에 add/commit/push.
- 인증: **deploy key(ed25519, write)** — gh CLI로 키 등록 + private key를 FD5to6 secret `REPORTS_DEPLOY_KEY`로. PAT 불필요, 권한 범위가 reports repo 1개로 최소.
- 로컬: `docs/reports/`를 reports repo의 clone으로 교체(부모 repo에선 gitignore). `scripts/run.sh`에 pull 1줄 추가.

## 1. 외부 동작 가정
- `actions/checkout@v4`의 `repository:`+`ssh-key:`+`path:` 조합이 sub-path checkout + push 자격(persist-credentials) 유지 → 공식 문서화된 동작(사실). [실환경 확인: 첫 run]
- deploy key(write)로 해당 repo push 가능, `gh repo deploy-key add --allow-write`·`gh secret set` 모두 현재 gh 인증으로 가능 → gh 권한 = repo admin(사실).
- private repo는 GH Pages와 무관 — Pages는 main repo(web/desk artifacts)만 사용(사실, deploy-*.yml 확인).
- 가드 step은 checkout 이후 파일 존재 검사라 위치 무관(사실, yml 확인).
- ⚠️ **기존 public 히스토리의 리포트는 남음** — HEAD에서 제거해도 과거 커밋으로 열람 가능. 완전 제거는 히스토리 재작성(별도 결정 필요, 이번 범위 밖 — 사용자에게 명시 보고).

## 2. 과거 부류 (errors.md)
- "git push 파이프가 실패 가림"(6/10) → 커밋 step은 파이프 없이 exit code 그대로.
- "배포 시각↔push 시각 미확인" 부류 → 전환 후 **실제 run의 reports repo 커밋 존재**로 검증(오늘 밤 US 라이브).
- 체인/가드 멱등 설계(6/10~) → 가드 로직 무수정이므로 멱등성 불변.

## 3. 추론 가능 vs 실환경 전용
- 머리로 거름: 경로 불변 전략의 파급 0(프롬프트·가드·weekly Read 전부 상대경로 docs/reports), 커밋 step의 대상 repo만 변경, 로컬 nested clone은 부모 gitignore로 충돌 없음.
- 실환경에서만: checkout ssh-key push 실동작, deploy key 권한 → **dry_run(checkout+생성 검증) + 오늘 밤 US 라이브(커밋 검증)**.
