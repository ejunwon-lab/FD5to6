## 2. 과거 부류 매칭 (errors.md)

- **Cloudflare PoP IP → GAS 403 (2026-05-31, 영구 불가 확정)** + **routine 환경 → GAS/Worker/Yahoo 차단 (2026-06-02~03)**: *DC IP에 대한 anti-abuse 차단은 코드·헤더로 우회 불가, 인프라 선택으로만 회피* — 이번 설계의 핵심 결정(비인증 스크래핑 → **인증 공식 API(KIS)**)이 이 교훈의 직접 적용. Naver를 GAS 프록시로 우회하는 안은 같은 부류의 재발 위험이 있어 **뉴스 한정 + 차단 시 degrade 명시**로 제한.
- **"로컬 curl 통과"로 실환경 판정한 오진단 (routine 사건에서 반복)**: 로컬 200은 Naver 정상의 증거일 뿐 러너 상태의 증거가 아님 → 러너 실측(A-1)을 구현 착수 조건으로 못박음.
- **"Version N already deployed" 착시**: 배포 후 반드시 배포 버전·시각을 API로 확인(가정 8 절차에 포함).
- **getUi() in 에디터 실행 함수**: 신규 GAS 코드는 Logger.log만 사용.
- **clasp push 금지**: push_safe.py 경유 (Secret.js 보호).

## 3. 추론 가능 vs 실환경 전용

**머리로 거름 (여기서 확정)**:
- doPost 분기 추가의 라우팅 안전성 (early return 구조)
- secret 게이트: `marketSnapshot`·`diagMarket` 모두 `TG_WEBHOOK_SECRET` 검증 (기존 `portfolioMetrics` 패턴 복제) — 비인증 노출 없음
- 응답에 원화 절대액 미포함 원칙 유지 (기존 portfolioMetrics 규칙 승계)
- weekly pre-fetch JSON은 `/tmp` 경로 → commit 대상( docs/reports/ ) 밖 — 리포트 커밋에 딸려 들어가지 않음
- KIS 호출 시각(17:05 KST)은 장 마감 후 — 당일 확정치 조회 가능

**실환경에서만 (스모크/실측으로만 판정)**:
- 러너→Naver·인포맥스·한경 실제 HTTP 코드/본문 (가정 1·2) → **A-1 진단 워크플로**
- GAS→Naver 허용 여부 (가정 3) → **GAS diagMarket 실측**
- KIS 업종/투자자 TR의 정확한 id·파라미터·응답 스키마 (가정 4·5) → **GAS 시험 호출** (문서 기억만으로 코딩 확정 금지 — 시험 호출 응답을 보고 파싱 작성)
- 전환 후 리포트 품질 (거래대금 숫자·업종 섹션 복원) → **dry_run 게이트**

## 4. 검증 방법 (사전 정의)

**자동 (Claude)**:
1. A-1 진단: `diag-egress.yml` workflow_dispatch 1회 → 로그에서 소스별 `HTTP <code> + 본문 head` — 예상: Naver 403 또는 200+차단페이지, Yahoo 200(대조군)
2. GAS 스모크: `diagMarket` action curl(로컬) → KIS 업종·투자자·지수 응답 JSON 필드 실물 확인 / Naver fetch 응답 코드
3. push_safe `node --check` 통과 + Apps Script API로 배포 3개 버전·시각 확인 (push 시각보다 이후)
4. `marketSnapshot` curl → walk-through: KOSPI 지수·등락%·거래대금(조 단위 환산)·업종 상위/하위 3·보유 5종 외인/기관 순매수 필드가 전부 숫자로 존재
5. KR 리포트 dry_run dispatch → 리포트 본문에 ① 거래대금 "—" 아님 ② 업종 강세/약세 섹션 실데이터 ③ "Naver 차단 N일차" 문구 소멸
6. weekly dry_run → dailyReturns가 "역산 추산" 아닌 GAS 수치로 표기, pre-fetch step 로그에 HTTP 200

**사용자**:
- 다음 거래일 17:05 텔레그램 KR 리포트에서 **시장 스냅샷 표의 거래대금**이 숫자(예: "12.3조")로, **자금 흐름 섹션**이 업종 실데이터로 표시되는지 확인 → 예상: "—(Naver 차단)" 문구 완전 소멸
