## 1. 외부 동작 가정 + 근거

- **web·web-desk·iOS 모두 `scripts.run` + `devMode: true`(HEAD 실행)** → push_safe push만으로 즉시 반영, 재배포 불필요 → 근거: `web-desk/src/api/gasApi.ts:17`, `web/src/api/gasApi.ts:17`, `ios2/Sources/Core/ScriptAPIService.swift:76` 모두 `devMode: true` 확인.
- **Telegram webhook(doPost)은 버전 고정 웹앱 배포 경로** → `_tgFormatPnL`이 `totalProfitRate` 사용(Telegram.js:110) → 변경 B 반영엔 재배포 필요 → `scripts/gas_redeploy.py`로 처리 (memory `reference_gas_redeploy_via_api`). 트리거(`tgPushPnL`)는 HEAD라 push만으로 반영.
- **추이 기록 U~AD 스키마 불변** — 읽기 전용 변경(윈도만 확장). 쓰기 경로(Trend.js) 무변경.
- **entries는 이미 거래일 필터 통과본** (`_isTradingDateStr`, errors.md 2026-06-21 v24 계열) → "180거래일" 산술 성립.
- 페이로드: 최악(11월 말) ≈ 245 entries ≈ 11KB — scripts.run 응답 한도 문제없음.

## 2. 과거 부류 (errors.md)

- **`slice(-N)`은 "최근 N행"이지 "최근 N거래일" 아님** (2026-06-21 d5 왜곡) → 이번 A·D가 정확히 같은 부류. A는 진입점 거래일 필터가 이미 있어 행=거래일 성립, D는 slice를 날짜 산술로 교체해 부류 자체 제거.
- **AD2 stale (2026-06-10)** — AD2 우선 로직은 건드리지 않음(무변경).
- **% 문자열 비대칭 (2026-07-16)** — % 쓰기/읽기 경로 무변경.

## 3. 추론 가능 vs 실환경 전용

- **머리로 거름**: 윈도 산식(min/max 클램프), 분모 합산(cfBuy 정의 = Trend.js confirmedBuy와 동일 컬럼 idx 10), 히트맵 첫 달 제외 로직, 날짜 cutoff 필터 — 전부 순수 로직, walk-through로 검증.
- **실환경에서만**: ① 배포 후 실제 응답 entries 첫 날짜가 전년 12/1 이전인지 ② 실데이터에서 totalProfitRate 하락 폭 — 이 맥에 .env 시크릿 없어 API 직접 실측 불가 → **사용자 확인 항목으로 이관** (데스크/웹 화면 값).
