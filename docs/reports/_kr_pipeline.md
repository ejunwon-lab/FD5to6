# KR 기회 파이프라인 — 상태 파일 (에이전트가 매일 갱신)

> KR 리포트 에이전트가 **리포트 작성 시작 시 Read → 오늘 관측 반영 → 이 파일을 다시 Write**.
> 목적: 기회 후보를 하루 노이즈가 아닌 **여러 날 누적/승급**으로 발굴. 커밋되어 다음날 이어짐.
> (dry_run에선 커밋 안 됨 = 테스트 1회성. 실제 스케줄 run에서만 누적.)

## 갱신 규칙 (에이전트용)
- **최근 강세 섹터**: 오늘 업종 강세 상위를 추가, 최대 7일 보관. 며칠 연속 등장 = 추세 강도↑.
- **후보 상태 승급**: 신규(1일) → 관찰(2~3일 지속) → 연구(3일+ 지속 & 유동성·내 포트 분산 적합) → (추세 꺾이면) 제거.
- **종목 자동선별 (Phase 2 활성, 2026-06-10~)**: 연구 후보 섹터는 Naver 업종 상세(`sise_group_detail`) 구성종목 표 상위 2~3을 실제 선별(프롬프트 2-D ③). fetch 실패 시에만 지식 기반 대표주 `〔후보·미검증〕`으로 degrade. 등락 수치는 fetch한 값만.
