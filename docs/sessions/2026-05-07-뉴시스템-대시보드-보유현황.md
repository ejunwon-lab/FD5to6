# 2026-05-07 — 뉴시스템 대시보드·보유현황 개선

## 완료 작업

### 1. 펀드/예금/보험 수동 입력값 리셋 버그 수정 (NewSystem.js)
- **증상**: 업데이트 실행 시 펀드·예금의 평가금액이 0으로 초기화됨
- **원인**: 백업이 `수동평가금액`(13열)만 저장했고, 사용자가 `현재단가`·`평가금액` 열에 직접 입력한 값은 누락
- **수정**: KIS_SKIP(펀드/예금/보험) 항목에 한해 현재단가(9열)·평가금액(10열)·수동평가금액(13열) 세 열 모두 백업·복원
- **레이아웃 감지**: 보유기간 열 추가 전(14열) / 후(15열) 자동 감지로 마이그레이션 안전 처리

### 2. 대시보드 카드 재구성 (Dashboard.js)
- **1행**: 총 매입금액 | 총 평가금액 | 총 수익 | 오늘의 수익
- **2행**: USD/GBP 환율 | 승률 | 확정 수익 | 운용 손익
- **오늘의 수익**: `*현재가_이력*` 마지막 2행 가격 차이 × 보유수량 계산. 데이터 2행 미만이면 `─` 표시
- **환율**: 투자수익 트래커 Named Range(`FX_USD`, `FX_GBP`)에서 읽음

### 3. 보유현황 보유기간 열 추가 (NewSystem.js)
- 위치: 계좌(5열)와 보유수량(6열) 사이에 삽입 → 6번째 열
- 형식: `X년 Y개월 Z일` (0값 생략, 예: `2년 3개월 15일`, `6개월 2일`, `45일`)
- 계산 기준: 최초 매수일 (전량 매도 후 재매수 시 날짜 리셋)
- 전체 인덱스 shift: Dashboard.js 포함 모든 참조 업데이트 완료
