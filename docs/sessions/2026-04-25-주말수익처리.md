# 2026-04-25 주말 수익 처리 & 메모리 정리

## 작업 내용

### 1. iOS 메모리 폴더 통합
- `FD5to6-ios` 프로젝트 메모리 파일 전체 삭제
- `FD5to6` 레벨 메모리만 유지

### 2. 대시보드 "오늘의 수익" 주말 처리

#### iOS (DashboardView.swift)
- `effectiveTradingDateString` computed property 추가
  - 오늘이 토(6) → -1일, 일(1) → -2일로 금요일 날짜 계산
  - "yyyy년 M월 d일 EEEE" 형식으로 "오늘의 수익" 아래 표시

#### GAS (MobileAPI.js)

**AH2:AI2 — 마지막 거래일 diff 캐시**
- `_logToTrendSheetLite`: 평일(월~금)에만 AH2:AI2에 diffProfit/diffRate 추가 기록
- AE2:AF2는 주말 포함 항상 갱신 (월요일 계산용)
- `_buildPortfolioJSON`: 주말이면 AH2:AI2, 평일이면 AE2:AF2 사용
  - AH2 비어있으면 AE2 fallback (초기 배포 대응)

**`initWeekdayDiffCache` 함수 추가**
- Section C 히스토리에서 마지막 평일 행 역순 탐색
- 히스토리 없으면 AE2:AF2 fallback (경고 출력)
- GAS 에디터에서 1회 실행용

#### 초기값 설정
- 토요일에 이미 앱 업데이트를 눌러 AE2가 토요일 값으로 덮어씌워진 상태
- AH2:AI2에 금요일 값을 수동 입력으로 해결

## 결과
- 주말에 앱 열면 금요일 수익 금액 + "2026년 4월 24일 금요일" 날짜 표시
- 시트 AE2는 주말에도 정상 갱신 → 월요일 계산 정상
- 다음 주 월요일 평일 업데이트부터 AH2:AI2 자동 갱신
