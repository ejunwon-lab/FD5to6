# 2026-05-13 신시스템 안정화 + 웹앱 신시스템 전환

## 주제
- 카톡 자동입력 시도 (구시스템 진행 → 롤백)
- broker 표기 통일 (`미래애셋` → `미래에셋`)
- 신시스템 추이 기록 다중 버그 수정
- 신시스템 사용성 (onOpen·setActiveSheet)
- 웹앱을 신시스템 GAS로 전환
- 9개 이슈 일괄 점검 + 수정

## 변경 사항

### 1. 카톡 자동입력 (실험 → 롤백)
- 구시스템 GAS에 `KakaoFill.js` + `Session.doPost` 추가, KODEX AI전력핵심설비 매도 100주 한 건 양쪽 시트 입력 성공
- 사용자 결정: 구시스템 [입력] 버튼 연결 문제로 구시스템 측 자동입력 보류, 모두 롤백
- `KakaoFill.js` 삭제, `Session.js` doPost·kakao action 제거, Web App 새 버전 재배포
- 메모리 3개(`project_kakao_workflow`, `reference_kakao_account_map`, `reference_kakao_endpoint`) 삭제

### 2. broker 표기 통일
- 신시스템 원장에 `미래에셋투자증권`(에)·`미래애셋투자증권`(애) 혼재 → `미래에셋투자증권`(에)으로 통일
- 구·신 양쪽 GAS에 `MigrateBroker.js` 임시 함수 추가, `createTextFinder().matchEntireCell(true)` 일괄 치환
- 구시스템 거래입력 시트 broker 드롭다운 옵션도 수정 (`TransactionSheet.js`)

### 3. 신시스템 *추이 기록* 다중 버그 수정 (`NewSystem.js _updateNewTrend`)
- **Section C writeRow** : `sheet.getLastRow()` 사용 → 5/11~5/13 행이 1500대 행에 떨어져 기록되던 버그. writeCol(날짜 열) 기준 last+1 로 변경
- **cfProfit 두 배** : 실현손익 필터 `r[2] !== '합계'` → `r[0] !== '합계'` (합계행 r[0]='합계'/r[2]=''이라 r[2] 기준 필터는 통과해버림)
- **fmtPct** : 문자열 `'12.34%'` → 숫자 `12.34` 반환 + 모든 섹션에 `setNumberFormat('0.00"%"')` / `#,##0` 적용
- **prevTotProfit** : row 2 스냅샷 대신 last data row(today 제외) 의 AD 컬럼에서 직접 계산 → row 2 오염되어도 안전
- `FixTrendLayout.js` 임시 함수로 기존 데이터 정리 (Section C 327행 5~331행에 재배치 + 5/11~5/13 cfProfit 절반 보정 + 일괄 포맷 + row 2 동기화)

### 4. 신시스템 사용성 (`Main.js`, `Code.js`, `Dashboard.js`, `AnalysisDashboard.js`)
- onOpen 중복 제거 (Code.js와 Main.js 두 파일에 동일 함수 있어 Code.js의 대시보드 자동 활성화가 덮어써짐) → Main.js로 통합
- `buildDashboard` / `buildAnalysisDashboard` 끝의 `ss.setActiveSheet(...)` 제거 — 전체 업데이트 중 시트가 자동으로 바뀌는 UX 문제 해결

### 5. 웹앱 → 신시스템 전환 (`web/src/api/gasApi.ts`)
- SCRIPT_ID `12MAcPpo...`(구) → `1DC8llpW...`(신)
- 7개 함수 매핑: `mobile*` → `newMobile*`
- 응답 구조 호환 확인 (PortfolioResponse / IndicatorsResponse / TrendHistoryResponse 동일)

### 6. 웹앱 portfolio state lifting (`App.tsx`, `DashboardPage`, `HoldingsPage`, `AnalysisPage`)
- 각 페이지가 자체 fetch → App 레벨 portfolio state 공유로 변경
- 원인: 한 페이지에서 업데이트해도 다른 탭은 mount 시 1회 fetch 한 옛 데이터 그대로
- 이제 업데이트 한 번 → 모든 탭 자동 반영

### 7. 신시스템 GAS 모든 진입점에 LockService 적용
- 모바일 API 3개(`newMobileUpdateCurrentPrice`/`UpdateHistory`/`UpdateAll`): 자체 lock
- 메뉴/트리거 진입점은 `_withUpdateLock(label, fn)` helper 적용 — `menuUpdateAll` / `menuUpdatePricesOnly` / `menuUpdateHistory` / `scheduledHistoryUpdate` / `scheduledDailyUpdate`
- 내부 함수(`updateAllNew` 등)에는 lock 없음 (중첩 lock 회피)
- 메뉴 항목 `'updateAllNew'` → `'menuUpdateAll'` 로 변경

### 8. ProfitHistoryChart lifting + visibilitychange (`App.tsx`)
- ProfitHistoryChart 자체 fetch 제거 → App 레벨 history state 공유
- `visibilitychange` 핸들러 추가 → 아이폰 화면 켜기/탭 복귀 시 portfolio + history 자동 새로고침
- `runUpdate` 종료 후 `fetchHistory` 호출 → 업데이트 시 차트도 자동 갱신
