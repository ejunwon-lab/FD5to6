### 2. iOS 번들 ID 불일치 에러 해결 (0xe8008016)

**증상**: 다른 폰에 설치 시 "The executable was signed with invalid entitlements" 에러

**원인**:
- `project.yml`의 `PRODUCT_BUNDLE_IDENTIFIER`를 `com.junwon.fd5to6finance`로 변경 (기존 `com.jun`이 다른 계정에 등록됨)
- 그런데 `Sources/Info.plist`의 `CFBundleIdentifier`는 `com.jw.fd5to6finance`로 하드코딩
- `project.yml`의 `info.properties`에도 `CFBundleIdentifier: com.jw.fd5to6finance` 하드코딩
- 코드서명(`com.junwon`) ≠ Info.plist(`com.jw`) → 불일치

**해결**:
- `Sources/Info.plist`의 `CFBundleIdentifier`를 `$(PRODUCT_BUNDLE_IDENTIFIER)` 변수로 변경
- `project.yml`의 `info.properties`에서 `CFBundleIdentifier` 항목 제거
- XcodeGen 재생성

**교훈**: `CFBundleIdentifier`는 반드시 `$(PRODUCT_BUNDLE_IDENTIFIER)` 변수로 지정. 번들 ID는 `project.yml`의 `PRODUCT_BUNDLE_IDENTIFIER` 한 곳만 관리.

**확정 번들 ID**: `com.junwon.fd5to6finance` (이후 변경 불필요)
