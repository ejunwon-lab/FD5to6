## 완료 작업

### 1. CLAUDE.md 경로 버그 수정
- `실행@`·새 컴퓨터 세팅의 memory 경로가 `Claude-2026-mini-FD5to6`(존재하지 않는 `_mini`)로
  잘못 적혀 있어 그대로 실행 시 엉뚱한 디렉토리로 복사됨 → `_mini` 전부 제거
- GAS 배포 경로 `apps-script/push_safe.py`(v1) → `apps-script-v2/push_safe.py`(활성)

### 2. 세션 키워드의 기계적 단계를 스크립트화
- 문제: `저장!`·`실행@`는 CLAUDE.md 산문을 Claude가 매번 해석·실행 → git push 누락 등
  단계 빠뜨림 위험 (언어 모델은 결정론적 실행이 아님)
- 해결: 기계적 단계를 스크립트로 분리
  - `scripts/save.sh "<커밋메시지>"` — memory 미러 + git add/commit/push
  - `scripts/run.sh` — git pull + memory 복원 + 전역 CLAUDE.md
  - 판단 단계(세션 문서 작성·현황 요약·커밋 메시지)만 Claude가 수행
  - memory 경로 슬러그는 `pwd` 기반 동적 계산 (폴더 이동에도 견딤)
- CLAUDE.md의 `저장!`·`실행@` 절차를 스크립트 호출로 갱신

### 3. 코드 재독 방지 장치
- `docs/code-map.md`에 **증상 → 위치 인덱스** 표 추가 — "X가 안 나와" → 파일·함수 직행
- `docs/api-reference.md` v2 기준 전면 재작성 — `newMobile*` 응답 전 필드 + 의미 +
  계산 위치 (기존엔 구시스템 v1 내용이었음)
- CLAUDE.md에 "버그 오면 코드 전에 errors.md → 증상 인덱스 → api-reference 먼저" 규칙
- (보류) GAS 파일 상단 함수 색인 주석 — code-map.md와 중복·이중 유지보수라 미실행
