# 2026-06-12 — 카톡 매매 알림 → GAS 거래_원장 자동 기록

## 목표
증권사(미래에셋/삼성) 카톡 체결 알림을 붙여넣으면 Claude가 파싱→확인→*거래_원장*에 1행 기록. 이후 `updatePositionFromLedger`가 보유현황·실현손익·종목지표·대시보드 전부 연쇄 → **원장이 단일 진실원천**.

## 설계 (/design-check 통과)
`docs/plans/2026-06-11-카톡매매-원장자동기록.md`. 핵심은 "로컬에서 시트에 쓰는 경로" 확정:
- ❌ 직접 Sheets API write(토큰에 spreadsheets 스코프 없음) / `scripts.run`(403, 스코프 부족)
- ✅ **doPost 웹앱 POST** — 코드는 push_safe로 배포, URL은 deployments API로 자동 획득, secret은 `.env`(gitignore)
- ✅ 버전 고정 배포 갱신은 `scripts/gas_redeploy.py`로 자율(메모리 `reference_gas_redeploy_via_api`)

## 구현
- **GAS** (`apps-script-v2/`):
  - `Main.js` doPost에 `action=addTrade` 분기.
  - `NewSystem.js` `_handleAddTradePost`(secret 검증) + `_appendTradeRow(payload)` — 정규화(증권사 별칭·코드 A제거·날짜 기본 오늘·금액 검산) → **멱등 체크**(주문번호+거래 동일 fill) → 매도 분류 *보유현황* 룩업(코드+증권사+계좌) → 원장 append → `updatePositionFromLedger` → `{beforeQty,afterQty}`.
- **로컬 헬퍼** (`scripts/`):
  - `gas_redeploy.py "<desc>" [marker]` — HEAD 새 버전 + 고정 WEB_APP 배포 갱신 + 마커 검증.
  - `post_trade.py '<json>' [--dummy]` — URL 자동·secret(.env) POST.
- **memory**: `reference_kakao_account_map.md`(마스킹→실계좌+수수료규칙). `037-99-****97`→미래에셋투자증권/종합_랩/무수수료.

## 스모크 중 발견·수정한 버그 (errors.md 2026-06-12)
1. 멱등 체크가 분류 룩업 뒤라 전량매도 후 중복 POST가 `category_required`로 빠짐 → 멱등을 **앞으로** 이동.
2. 원장 날짜가 Date 값이라 `String(r[0])===dateStr` 실패 → Date/문자열 양쪽 견고화.
→ v16 재배포 후 `already:true`(중복 0) 확인.
