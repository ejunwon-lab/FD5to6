# 2026-05-17 — 변동 라벨 비거래일 버그 수정 + 수익 기준일 표시

## 배경
일요일(비거래일)에 iOS·웹 대시보드가 "오늘 수익"으로 표시되고, 잠깐 정상이다가
다시 "오늘"로 바뀌는 현상 발생. 5월 15일 금요일 수익을 "최근 수익"으로 보여줘야 함.

## 완료 작업

### 1. "오늘 수익" 라벨 오표시 버그 (errors.md 2026-05-17 참조)
- **원인**: `updateNewPriceHistory()`(NewSystem.js)가 거래일 여부 확인 없이
  *현재가_이력*에 오늘 날짜 행을 추가 → 주말에 업데이트 시 비거래일 날짜 행 누적
  → `priceAsOfDate`(마지막 행 날짜)가 비거래일 날짜가 되어 `decideChangeLabel`이
  "오늘" 반환. `_mCalcExtras` 변동 계산도 오염. "작동하다 바뀜" = 캐시(정상) → fresh fetch(오염)
- **해결 (A+B+C)**:
  - A. `updateNewPriceHistory`에 `_mIsTradingDay()` 가드 — 비거래일엔 행 미기록
  - B. `priceAsOfDate`는 마지막 *거래일* 행 역방향 스캔, `_mCalcExtras`는 비거래일 행 필터링
       (`_isTradingDateStr` 헬퍼 신규)
  - C. 클라이언트 `decideChangeLabel`(changeLabel.ts / ChangeLabel.swift)이
       비거래일이면 priceAsOfDate와 무관하게 "최근" 우선 판정
- GAS v2 배포 완료 → 백엔드만으로 iOS·웹 즉시 정상화

### 2. 수익 카드 맨 아래 줄 = 수익 기준일 표시
- **문제**: 웹앱이 `updatedAt`(API 호출 시각, 2026-05-17 13:18)을 표시. 수익 기준일이어야 함
- **해결**: `formatPriceAsOfDate()` 헬퍼 신규 ("2026-05-15" → "2026년 5월 15일 금요일")
  - Web: `changeLabel.ts` + `DashboardPage.tsx` — `priceAsOfDate` 기준 표시
  - iOS: `ChangeLabel.swift` + `DashboardView.swift` — 요일 역산 휴리스틱
    `effectiveTradingDateString` 제거, `asOfDateString`(priceAsOfDate 기준)으로 교체.
    미사용/공휴일 미반영 헬퍼 `isBeforeMarketWindow`/`showPrevDayProfit`/`showRecentProfit` 제거
