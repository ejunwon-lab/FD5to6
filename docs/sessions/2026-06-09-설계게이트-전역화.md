# 2026-06-09 — 코딩 전 설계 게이트(/design-check) 구축 + 전역 동기화

## 배경
텔레그램 푸시 검증(같은 날 `2026-06-09-텔레그램푸시-KR데이터-검증.md`)에서 errors.md 텔레그램 관련 에러 5건 확인. **그 중 4건(302 redirect·MYSELF 권한·best-effort 스케줄러·update_id 루프)이 코딩 전 추론·문서로 알 수 있었던 것**이었음. 사후 검증(스모크·모니터링)보다 **코딩 전 사고가 토큰·신뢰 둘 다 싸다**(재작업 루프 1회 ≫ 설계 1회)는 캡틴 통찰에 따라, 코딩 전 게이트를 하네스 구조로 구축.

## 만든 것

### 1. 코딩 전 설계 게이트
- **전역 CLAUDE.md "변경 전 설계 절차"** — 위험 변경 트리거(배포 서버코드·CI/스케줄 파이프라인·권한·외부 API) + 4항목 체크리스트(외부 동작 가정·과거 에러 검색·추론 가능 vs 실환경 전용 분리·검증 방법).
- **`/design-check` skill** — 위 절차를 ultrathink로 실행하는 재사용 절차. 산출물은 `docs/plans/YYYY-MM-DD-주제.md`.
- **프로젝트 CLAUDE.md** — 원칙은 전역 참조, 이 repo의 구체 트리거 경로(apps-script-v2·workflows·appsscript.json·KIS/Naver)만 매핑.
- **2단계 전략**: 현재 규칙+skill(자율). 다음 위험 변경에서 효과(코딩 전 에러 차단) 증명 후 PreToolUse hook으로 강제.

### 2. 전역 설정 두 머신 동기화 (repo = SSOT)
- `~/.claude/`는 git 밖 → 동기화 유일 경로 = **repo 원본 → run.sh가 복사**.
- 발견: `config/global-claude.md`가 옛 stale(44줄, 슬러그 규칙도 틀림), 실제 `~/.claude/CLAUDE.md`(58줄)가 최신. run.sh는 "없을 때만" 복사라 갱신 안 됨. 전역 skill 동기화 경로 자체 없음.
- 수정: ① 맥미니 58줄본 → `config/global-claude.md`로 끌어올려 SSOT 정립 + 설계절차 추가 ② `config/skills/design-check/SKILL.md` 원본화(일반화) ③ `run.sh` — 전역 CLAUDE.md **항상 동기화 + 첫 1회 `.bak` 백업(머신 고유 내용 손실 방지)** + `config/skills/. → ~/.claude/skills/` 복사.
- 규칙: 이후 `~/.claude/`는 직접 수정 금지(다음 실행@가 덮어씀). 전역 변경은 repo `config/`에서만.
