# 2026-04-26 iOS 빌드 에러 해결 (타기기 설치)

## 작업 내용

Sooen Chang의 iPhone(iPhone 14, iOS 18.6)에 앱 설치 시 발생한 빌드/서명 에러 3종 해결.

### 1. 번들 ID 변경
- `com.jw.fd5to6finance` → `com.jun.fd5to6finance`
- 기존 번들 ID가 다른 팀/계정에 이미 등록되어 있어 Sooen Chang 팀으로 등록 불가
- 수정 파일: `ios/FD5to6Finance.xcodeproj/project.pbxproj` (Debug/Release 양쪽)

### 2. BackgroundNetworkSession 식별자 수정
- `com.jw.fd5to6finance.gas-update` → `com.jun.fd5to6finance.gas-update`
- 수정 파일: `ios/Sources/Core/BackgroundNetworkSession.swift`

### 3. Info.plist 번들 ID 수정 (핵심 원인)
- `CFBundleIdentifier`가 `com.jw.fd5to6finance`로 하드코딩되어 있었음
- 코드사인은 `com.jun.fd5to6finance`인데 Info.plist는 `com.jw.fd5to6finance` → 불일치로 `0xe8008016` 에러
- `$(PRODUCT_BUNDLE_IDENTIFIER)`로 변경
- 수정 파일: `ios/Sources/Info.plist`

### 4. Background Modes 추가 (Xcode에서 직접)
- Xcode → Signing & Capabilities → Background Modes 추가
- fetch, processing 체크
- Info.plist에 `UIBackgroundModes` 자동 추가됨

## 결과
- Sooen Chang 팀(Z9U78H6GDL)으로 정상 서명
- 기기 등록 및 설치 성공
- 폰에서 "신뢰할 수 없는 개발자" → 설정에서 신뢰 처리로 실행 완료
- 앱 용량: Debug 빌드 기준 5.1MB
