## 변경 사항

### 1. 카톡 자동입력 (신시스템 전용)
- `apps-script/KakaoFill.js` 신규: 구시스템 GAS Web App 경유로 신시스템 *거래_입력폼* 자동 채움
- `kakaoLookupByName(name, code)`, `kakaoSnapshot()`, `kakaoFillForms(payload)`
- `Session.js` doGet/doPost 에 action 라우팅 추가
- 이전과 달리 구시스템 시트는 안 건드림

### 2. 종목상태 이력 시트
- 신시스템 `*종목상태_이력*` 시트 자동 생성 + 일자별 upsert
- `_appendStockStatusHistory(ss)` 추가, `updateNewCurrentPrice` 끝에 호출
- 컬럼: 날짜·종목코드·종목명·가격·등락·등락률·52주최고/최저·1M/3M/6M/1Y

### 3. isTradingDay 필드 추가 + 라벨 로직 통일 (신·구)
- `summary.isTradingDay` (시간 무관, 평일+공휴일 아님) 신규 필드
- 라벨 분기:
  - 비거래일 → "최근 수익"
  - 거래일 09:00 이전 → "전일 수익"
  - 거래일 09:00 이후 → "오늘의 수익"
- ios2 + 구 iOS + 웹앱 동일 로직

### 4. 데이터 정확도 (신시스템)
- **`_updatePriceHistory` upsert 버그 수정**: 같은 날짜 행 30개 중복 누적 → `_toDateStr` 헬퍼 + 중복 자동 정리
- **`dedupPriceHistory`** 수동 정리 함수 (1회 실행)
- **`*장기_가격_이력*` 시트 + `backfillLongTermHistory`**: KIS 주봉(1Y2M) + 일봉(4M) → 종목별 wide 포맷
- **`_calcReturnFromHistory`** 두 시트 통합 참조 (현재가_이력 우선)
- **신규 종목 자동 1Y 백필**: `_autoBackfillMissingCodes` — `updateNewStockHistory` 시작 + `addTransactionFromForm` 끝에 자동 호출
- **`buildDashboard`** 가 *종목상태* 직접 참조 — *보유현황* 갱신 안 해도 즉시 반영
