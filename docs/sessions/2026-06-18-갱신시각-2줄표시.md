# 2026-06-18 — 마지막 갱신시각 표시 개선 (웹·iOS) + 데이터 정확화(GAS)

## 요청
갱신하면 갱신 시각 표시, 날짜·시간 2줄, 폰트 2단계↑. 갱신 안 하고 기존값 가져오면 날짜·시간도 같이 가져오기.

## 핵심 발견
`newMobileGetPortfolio`의 `updatedAt`이 `new Date()`(응답 시각) → 단순 읽기도 항상 "지금"으로 표시되던 문제. 실제 데이터 갱신시각은 *대시보드* 2행(`buildDashboard`/`_writeDashboardStatus`가 기록)에 저장됨.

## 변경
- **GAS** `apps-script-v2/MobileAPI.js`: `_mLastUpdateAt(ss)` 추가 — *대시보드* 2행 정규식 파싱 → 저장 갱신시각(yyyy-MM-dd HH:mm). `newMobileGetPortfolio.updatedAt`이 이걸 사용. 갱신(updateAllNew→buildDashboard 재기록)=now, 단순읽기=직전 갱신시각. try/catch+now() 폴백(getPortfolio 깨질 수 없음). **v17 배포**(push_safe + gas_redeploy).
  - 클라이언트는 `scripts.run`(사용자 OAuth, spreadsheets 스코프)으로 호출 — 로컬 clasp 토큰은 스코프 부족이라 라이브 스모크 불가, 앱에서 확인.
- **웹** `web/src/utils/changeLabel.ts`: `splitUpdatedAt()` + 테스트 4건. IndicatorsPage 푸터 날짜/시간 **2줄**+`text-sm`. DashboardPage 기준일(priceAsOfDate) **유지** + "마지막 갱신/날짜/시간" 2줄 추가(text-sm). (vitest 13 통과, tsc 0)
- **iOS** `ios2/Sources/Shared/ChangeLabel.swift`: `splitUpdatedAt()`. DashboardView·IndicatorsView footer → 라벨/날짜/시간 3줄 + `.caption2→.footnote`.

## 폰트 단계
웹 `text-[10px]→text-sm`(14px), iOS `.caption2→.footnote`. 사용자 "제안대로" 승인.

## 배포 경로 (중요)
- GAS: push_safe + gas_redeploy로 직접 배포(즉시 라이브).
- 웹: **git push → `deploy-web.yml`(GitHub Pages) 자동 빌드**. push 전엔 옛 빌드라 변화 안 보임 → 하드 새로고침 필요.
- iOS: Xcode 빌드.

## 검증
- 웹 vitest 13/13, tsc 0. ChangeLabel.swift parse OK.
- 🔴 사용자: 웹 배포 후 하드새로고침 → 대시보드(기준일+마지막갱신 2줄)·지표(2줄) 14px. "갱신 안 함" 시 updatedAt이 직전 실제 갱신시각인지. iOS Xcode 빌드 확인.
