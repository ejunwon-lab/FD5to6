# 2026-07-23 (저녁) — web-desk 테마 시스템 (Modern/Terminal)

## 배경

사용자 피드백: 모바일에서 가독성이 떨어지고 폰트·컬러도 불만. 진단 결과 원인 4가지 —
① JetBrains Mono에 한글 글리프가 없어 한글만 시스템 폰트로 폴백(혼재), ② 주력 글자 크기 10~12px(데스크톱 밀도 기준), ③ 네온 원색(halation) + 보조 텍스트 저대비(`ink-faint` 2.59:1 — WCAG 미달), ④ CRT 스캔라인 노이즈.

사용자 결정: ①크기·대비 ②폰트 ③컬러 전부 개선하되 **테마로 만들어 스위치 가능하게**.

## 구현

- **테마 엔진**: 모든 색을 `web-desk/src/index.css`의 RGB 채널 변수(`--c-*`)로 이관, `tailwind.config.js`는 `rgb(var()/alpha)` 소비. `:root`=Terminal(기존 네온 그대로), `[data-theme='modern']`=개선판. `lib/settings.ts` `applyTheme()`가 `<html data-theme>` 스탬프 — 변수 캐스케이드라 리마운트 없이 즉시 전환.
- **Modern**: 본문 Pretendard Variable(dynamic subset CDN, 숫자는 `.tabular` tnum 유지) · 텍스트 토큰 전부 대비 ≥4.5:1 계산 확정 · 네온→소프트 팔레트 · 스캔라인 제거.
- **차트 시리즈 분리** (`--c-s1~s7`): 텍스트(대비 기준)와 면 채움(다크 밴드 OKLCH 0.48–0.67 기준)이 상충 → 분류 도넛은 dataviz 검증기 5체크 전부 통과한 전용 팔레트. Terminal 값은 기존 도넛 색 그대로.
- **모바일 크기**: `--fs-2xs/xxs/xs` 10/11/12 → <640px에서 11/12/13px (양 테마 공통).
- **차트 hex 청소**: 10개 컴포넌트의 하드코딩 hex(축·툴팁·스파크라인·마커)를 `rgb(var(--c-…))`로 일괄 교체. 이후 raw hex 금지 규칙 수립(decisions.md).
- **Settings**: "테마" 패널 추가(Modern/Terminal, 즉시 반영), 기본값 Modern. `desk_settings_v1`에 `theme` 필드 추가.
- 문서: code-map(테마 시스템·Settings 항목)·decisions(2026-07-23 설계 근거)·pending 갱신.
