### 4. `_updateNewTrend()` 전면 재작성 — 3섹션 구현

#### Section A: 업데이트별 추이 (B~L, 11열)
- 항상 append (날짜 중복 허용, 구시스템 동일)
- B2:L2 최신 스냅샷 항상 덮어쓰기
- 날짜 포맷: B=`yyyy-MM-dd (EEE)`, C=시간 별도 (구시스템 호환)
- 데이터: opNow(운용합계), pendNow(대기합계), sumNow(전체합계) + 각 변화량/률

#### Section B: 일별 추이 (N~S, 6열)
- upsert (N열 날짜 기준, 오늘 행 있으면 덮어쓰기)
- N2:S2 최신 스냅샷 항상 덮어쓰기
- bTodayRow 있으면 dTodayRow-1의 Q열로 prev 계산
- 데이터: opNow, pendNow, sumNow, 변화금액, 변화율

#### Section C: 수익 추이 (U~AF, 12열) — 기존 대비 개선
- upsert 탐색 시작: row 2 → **row 5** (row 2 스냅샷과 충돌 방지)
- prev 읽기 조건: `writeRow > 2` → **`writeRow > 5`**
- **U2:AF2 최신 스냅샷 추가** (기존에 없었음)
- AH/AI/AJ/AK: writeRow 기준 → **row 2** 기준으로 변경
- AJ/AK 백업 조건: `writeRow > lastRow` → **`prevU2Date !== today`** (구시스템 동일)

#### useOldCol 감지 로직 개선
- 기존: `lastCol >= 30` → Section A/B 추가 후 항상 true
- 수정: U열(col 21) 5~50행에 `/^\d{4}-\d{2}-\d{2}/` 정규식 매칭
