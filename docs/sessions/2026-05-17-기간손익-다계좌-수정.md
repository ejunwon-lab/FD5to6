## 완료 작업 (errors.md 2026-05-17 참조)

### 1. 다계좌 보유 종목 손익 누락
- **원인**: Dashboard.js `_calcExtraColumns()`의 `txByCode`/`qtyAtDate`가 *거래_원장*을
  종목코드 단위로만 합산. 가드 `qtyAtDate(code) !== curQty`에서 `curQty`는 *보유현황*
  행별 계좌별 수량 → 같은 종목을 2개+ 계좌 보유 시 영구 불일치 → `pnlAt` 항상 null
- **해결**: `txByKey`를 `code||증권사||계좌` 단위로 키잉, `qtyAtDate(key, …)`로 계좌별 판정

### 2. 랩 계좌 등 거래 잦은 계좌도 빈칸 → 정확한 기간 손익으로 교체
- 1번 수정 후에도 랩 계좌(미래애셋 종합_랩)는 기간 내 거래가 잦아 수량 변동 가드에 계속 걸림
- **해결**: `pnlAt`을 정확한 기간 손익 공식으로 교체, 수량 변동 가드 제거
  ```
  기간 손익 = 오늘 평가금액 − N일전 평가금액 − 기간 내 순매수금액
            = tPrice×curQty − pastPrice×pastQty − (Σ매수금액−Σ매도금액)
  ```
  - 수량 불변 종목은 결과 기존과 동일 (= 가격변동×수량)
  - 거래한 종목·신규 진입 종목도 올바른 값 산출
  - 원장 read 8→10열 확장 ('금액' 열 사용), `netInvestedSince()` 헬퍼 추가
