# 2026-07-17 — 제헌절 휴장일 반영 + 텔레그램 오발송 수리

## 문제
오늘(2026-07-17) 제헌절인데 KRX 휴장일임에도 `tgPushPnL` 장중 텔레그램 푸시가 계속 발송됨.

## 원인
- 제헌절이 2026-05-11 발효 법 개정으로 18년 만에 공휴일 재지정, **첫 적용일이 당일**.
- *휴장일* 시트의 마지막 동기화(2025-12월 `scheduledHolidaySync`)는 발효 이전 → 2026-07-17 행 없음.
- 재동기화해도 `HOLIDAY_NAMES` 화이트리스트(스승의날 사고 2026-05-17의 해결책)에 '제헌절'이 없어 걸러짐.
- → `_isKoreanHoliday(2026-07-17)`=false → `_mIsTradingDay`=true → 발송.

## 조치
1. **응급 정지 (12:35 KST)** — `gh workflow disable "Telegram Push (intraday P&L)"` + 실행 run 취소. (주의: disable은 신규 실행만 막음. 11:25 시작 in_progress run은 계속 poke했으나, 시트 갱신 후 전부 skip-holiday로 전환됨.)
2. **코드 수정 — Holidays.js 3곳** (설계: `docs/plans/2026-07-17-제헌절-휴장일-반영.md`):
   - `HOLIDAY_NAMES`에 '제헌절' 추가
   - KRX 고정휴장 루프에 `y>=2026 && !map['07-17']` → '제헌절' 고정 삽입 (캘린더 수록 여부 무관 보장, 5/1·12/31 패턴)
   - `_HOLIDAY_FALLBACK`에 '2026-07-17' 추가
3. **배포** — `push_safe.py`(11개 .js + appsscript, node --check 통과). 메뉴/트리거는 HEAD 실행이라 즉시 반영. 웹앱 버전 배포 불필요(판정 함수는 시트를 runtime 읽음).
4. **시트 갱신** — 사용자가 메뉴 🗓️ 휴장일 동기화 클릭 → *휴장일* 시트에 2026-07-17 행 확인 완료.
5. **workflow 재가동** — `gh workflow enable` + `gh workflow run`(dispatch 재시드). 월요일 정상 작동.

## 부수 확인 — Secret 원격 전용 설계 검증 (사용자 요청)
- push_safe 출력 `Protected on remote: none`을 보고 "Secret 원격에 없음"으로 오판 → 사용자 지적으로 재검증.
- 원인: push_safe `PROTECTED={"Secret"}`가 이름 `"Secret"`으로 찾는데 원격 실제 파일명은 `Secret.gs` → 보호 체크만 놓침. **파일은 merge 구조라 그대로 보존**(원격 전용 파일 미변경).
- 전수 검증 결과 — **Secret은 GAS(원격)에만 존재하는 설계 그대로 유지**:
  - 로컬 `apps-script-v2/`에 Secret 파일 없음
  - 원격 GAS에 `Secret.gs` 존재
  - `.gitignore:14` `apps-script-v2/Secret.js` 명시 제외
  - git 추적/히스토리에 실토큰 값 0건 (BOT_TOKEN 매치는 전부 변수명·프로퍼티 키 참조)
- 검사 중 🔴 실토큰 오탐 1회 — `grep | head` 구조에서 head가 빈 입력에도 성공 코드 반환 → `&& echo` 오발화. 파이프 없이 재확인해 0건 확정.

## 교훈
외부 세계의 목록(공휴일·수수료율)을 코드 화이트리스트로 박제하면 제도 변경 시 조용히 낡는다. 연중 공휴일 신설은 12월 동기화 주기로도 못 잡음 → 고정 날짜는 이름 매칭보다 코드 보장이 견고.
