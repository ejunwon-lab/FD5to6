# 2026-04-27 iOS 콜드런치 에러 해결

## 증상
앱 아이콘 탭 시 "Finance를 더 이상 사용할 수 없음 (확인)" 다이얼로그. Xcode 직접 실행은 정상, 앱 스위처 재개도 정상, 아이콘 콜드 런치만 실패.

## 원인 (복합)

1. **CODE_SIGN_STYLE = Automatic 누락** — 이전 커밋(d06e576)에서 XcodeGen 재생성 시 project.yml에 없던 설정이 pbxproj에서 삭제됨. Xcode 디버거 실행은 우회되지만 iOS 콜드 런치는 서명 검증 실패
2. **DEVELOPMENT_TEAM 불일치** — 이전 Mac 팀 ID(44DWWF283N)가 잔류. 이 Mac의 실제 인증서 팀 ID는 3N9UDPW4BP
3. **번들 ID 변경 후 구 앱 잔존 (실제 원인)** — `com.jun.fd5to6finance` → `com.junwon.fd5to6finance` 변경 시 Xcode는 새 앱만 설치하고 구 앱은 삭제하지 않음. 홈 화면에 두 개의 Finance 앱이 존재. 탭하던 아이콘이 만료된 구 앱이었음

## 해결
- 기기에서 구(만료) Finance 앱 직접 삭제
- project.yml에 `CODE_SIGN_STYLE: Automatic`, `CODE_SIGN_IDENTITY: "Apple Development"`, `DEVELOPMENT_TEAM: 3N9UDPW4BP` 추가
- project.pbxproj Debug/Release 두 곳에 동일하게 적용

## 교훈
- **번들 ID 변경 시 반드시 기기에서 구 앱 먼저 삭제 안내**
- XcodeGen 재생성 전 project.yml에 서명 관련 설정이 모두 명시되어 있는지 확인
