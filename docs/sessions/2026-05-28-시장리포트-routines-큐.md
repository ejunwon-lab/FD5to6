# 2026-05-28 시장 리포트 자동 발송 — Claude Routines + 시트 큐 + Telegram

## 배경

사용자 요청: **거래일 08:00에 전날 미국 시장 요약 + 17:00에 한국 시장 마감 요약**을 Telegram으로 자동 발송. 단순 지표 나열이 아니라 *전문 애널리스트 수준* — 시장 이슈·등락 원인·외인/기관/개인 수급·보유 종목 영향·섹터 분석 포함.

## 결정 과정

### 분석 엔진
- 단순 수치만으론 "왜 올랐나" 코멘트 불가 → **LLM 호출 필수** 판단
- 최초 옵션 비교: Claude API (월 ~$3) / Gemini API (무료) / Cloudflare Workers AI (저품질)
- 사용자가 **Claude Max 구독자** — API와 별도 채널이라 추가 결제 필요
- 사용자 제안: "Claude의 다른 기능으로 시트에 적고 GAS가 Telegram 발송"
- 사용자 확인: **Claude Routines — Max 무료 15개/일** → 채택. 우리 사용량 2개, 한도 13개 여유.

### 아키텍처
```
[claude.ai routine 08:00 / 17:00 KST] (Max 무료 한도 내)
   ↓ WebFetch로 외부 데이터 수집 (Yahoo, 네이버, Reuters 등)
   ↓ 분석 + 리포트 본문 생성 (Claude 자체)
   ↓ HTTPS POST → GAS Web App (action=addMarketReport, secret, type, title, body)
[GAS doPost]
   ↓ _tgHandleMarketReportPost → *시장리포트_큐* 시트에 "대기" 행 추가
[GAS 트리거 08:05 / 17:05]
   ↓ tgFlushReportQueue → "대기" 행 → tgSendMessage → "발송완료" 마킹
```

핵심: 시트가 **로그 + retry 큐** 역할. Routine 실패 시 빈 큐로 안전 종료, 발송 실패 시 에러 메시지 시트에 보존.

### MVP 단순화 결정
- 보유 종목 정보를 routine이 GAS에서 받아오려면 `doGet` 신설 또는 Google Sheets MCP 필요. 둘 다 불확정 영역.
- → **보유 종목 리스트는 routine 프롬프트에 정적 박음**. 종목 변경 시 routine 프롬프트 수정 (빈도 낮음 가정).
- 외부 시장 데이터는 routine이 WebFetch로 직접 수집 — GAS는 dispatcher 역할만.

## 구현 내용

### `apps-script-v2/Telegram.js`
- `TG_REPORT` 상수 추가 — 시트명·트리거 핸들러·상태(대기/발송완료/실패)·헤더 8컬럼
- `_tgEnsureReportQueueSheet()` — *시장리포트_큐* 시트 자동 생성/보장
- `_tgHandleMarketReportPost(e)` — doPost 핸들러. JSON/form-encoded 둘 다 지원, secret 검증(TG_WEBHOOK_SECRET 재사용), type(US/KR) 검증, 시트 appendRow
- `tgFlushReportQueue()` — 시트의 "대기" 행 일괄 발송, F~H열에 결과 마킹
- `tgSetupReportQueueTrigger()` / `tgDeleteReportQueueTrigger()` — 매일 08:05·17:05 KST 트리거 2개
- `tgFlushReportQueueNow()` — 메뉴 수동 발송용

### `apps-script-v2/Main.js`
- `doPost(e)` — `action=addMarketReport`면 `_tgHandleMarketReportPost`로 위임, 그 외는 기존 `handleTelegramWebhook` (JSON 본문에서도 action 추출)
- `onOpen()` — 유지보수 메뉴에 3개 추가:
  - 📊 시장 리포트 — 08:05·17:05 트리거 ON
  - 🗑️ 시장 리포트 트리거 OFF
  - 📤 시장 리포트 — 큐 즉시 발송

### `docs/market-report-routines.md` (신규)
- 셋업 가이드 (3 step): GAS 새 deployment / 시트 메뉴 트리거 / Routine 등록
