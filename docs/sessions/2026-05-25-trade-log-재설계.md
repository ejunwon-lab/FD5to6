# 2026-05-25 — Trade Log (ActivityPage) 전면 재설계

## 배경

다른 창에서 Indicators 카테고리 그룹화 작업 완료 후 (commit `0b5f679`),
이어서 Trade Log 점검 중 API surrogate 에러로 그 창 중단 → 이 창에서 이어감.

## 발견한 문제 (3가지)

### 1. 사일런트 버그 — 응답 키 미스매치

| 쪽 | 키 |
|---|---|
| GAS 응답 (`newMobileGetMonthlyRealized`) | `monthly` |
| 데스크 (`DataProvider.refreshRealized`) | `entries` |

→ `res.entries ?? []` 가 항상 빈 배열 → SAMPLE_REALIZED fallback만 표시.
사용자가 보던 표는 **실제 매매 데이터가 아니라 샘플**.

### 2. GAS endpoint가 월별 집계만 반환

실현손익 시트는 14컬럼 풀 데이터 (매도일·종목코드·종목명·분류·증권사·계좌·매도수량·매도단가·매도금액·평균매입단가·매입원가·수수료·실현손익·수익률) 보유.

그런데 endpoint는 `{month, count, winCount, profit, profitRate, winRate}` 6필드 집계만 반환.
→ 종목·매도일·증권사 등 *건별 회고*에 필요한 행 단위 정보가 데스크에 도달 못 함.

흥미로운 점: **데스크 타입(`MonthlyRealizedItem`)은 이미 "행 단위" 모양으로 설계**되어 있었음.
GAS만 잘못 구현된 케이스.

### 3. 표시 규칙 위반

- 테이블 첫 컬럼 "Symbol"(종목코드 메인) → memory `feedback_stock_name_primary` 위반
- 월별 막대 라벨 `+45만` 축약 → memory `feedback_number_display` 위반

## 변경 사항

### GAS — `apps-script-v2/MobileAPI.js`

함수명 유지 `newMobileGetMonthlyRealized`. 응답을 **행 단위 14필드**로 변경.

```json
{ "success": true, "entries": [{
  "date":"2026-05-20", "month":"2026-05",
  "code":"005930", "name":"삼성전자", "category":"한국주식",
  "broker":"미래에셋", "account":"종합_랩",
  "quantity":15, "sellPrice":75000, "sellAmount":1125000,
  "avgBuyPrice":60000, "buyCost":900000,
  "fee":1500, "profit":223500, "returnPct":24.8
}, ...] }
```

- 정렬: 매도일 내림차순
- KPI/월별 집계는 클라이언트에서 derive (`useMemo`)

### 데스크 — `web-desk/src/api/gasApi.ts`

`MonthlyRealizedItem` 14필드로 확장 (date·broker·account·sellPrice·avgBuyPrice·sellAmount·buyCost·fee·category 추가).

### 데스크 — `web-desk/src/components/activity/ActivityPage.tsx`

**KPI 5칸 → 6칸** (Total Fees 추가):
- Realized P&L · Win Trades · Loss Trades · Win Rate · Avg Trade · **Total Fees**
- Total Fees: 전체 수수료 합 + per-trade 평균 (sub 라인)

**테이블 컬럼 재구성** (6컬럼 → 9컬럼):
- Date · 종목(이름 메인+코드 보조) · 증권사·계좌 · 수량 · 매수가 · 매도가 · 수수료 · 손익 · 수익률
- min-width 680px → 960px

**표시 규칙 적용**:
- 종목명 `text-amber font-medium` (메인), 종목코드 `text-xxs text-ink-faint` (보조)
- 막대 라벨 `+45만` → `+₩450,000` 풀 (`fmtSignedKrw` helper)
- 모든 금액 `Math.round(n).toLocaleString()`

**Sample data 8건 새 스키마로 갱신** (broker·account·매수가·매도가·수수료 채워 fallback도 실제와 같은 모양).
