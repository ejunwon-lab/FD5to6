# 2026-05-08 뉴 시스템 버그 수정

## 작업 내용

### 1. 보유현황 시트 헤더 열 어긋남 수정 (NewSystem.gs)
- 원인: 시트는 14열(구형, 보유기간 없음)로 생성됐는데 코드는 15열(보유기간 포함)로 쓰고 있어서 열 어긋남 발생
- 수정: `updatePositionFromLedger` 실행 시 헤더 6번째 열이 '보유기간'이 아니면 자동으로 15열 헤더로 업데이트

### 2. KIS_SKIP 항목 보호 (NewSystem.gs)
- 원인: `updatePositionFromLedger`가 전체 시트를 지우고 다시 쓰는 구조라 수동 입력(펀드·예금·보험) 값이 유실되는 문제. 백업/복원 로직이 열 인덱스 불일치로 깨져 있었음
- 수정: 기존 `manualMap` 백업 방식 폐기 → KIS_SKIP 항목은 행 전체를 `skipRowMap`에 저장 후 그대로 복원. 보유기간만 최신화
- KIS_SKIP에 `기타` 분류 추가

### 3. 매수 수수료 원가 반영 (NewSystem.gs, Transaction.gs)
- 원인: 매수 수수료가 원가에 포함되지 않아 손익 계산 오류
- NewSystem.gs: `p.totalCost += a + fee`로 수정
- Transaction.gs `handleBuy`: 기존 포지션 추가 시 `(existingQty×existingPrice + qty×price + fee) / newQty`, 신규 시 `(qty×price + fee) / qty`

### 4. 오늘의 수익 계산 안 되는 버그 (Dashboard.gs, NewSystem.gs)
- 원인: `*현재가_이력*` 날짜가 Date 객체로 저장되는데 `String(date).slice(0,10)`으로 읽어 "Fri May 08" 형태가 되어 날짜 비교 항상 실패
- 수정: `dates[i][0] instanceof Date ? Utilities.formatDate(...) : String(...).slice(0,10)` 으로 변경
- 부작용: upsert 체크도 같은 버그라 매번 중복 행 추가되고 있었음 → 동시 수정

### 5. 확정 수익 잘못 읽는 버그 (Trend.gs)
- 원인: 역방향 탐색으로 K·V·X를 읽는 로직에서 X(STATUS_Y1) 값이 합계행보다 아래 데이터 행에서 먼저 잡힘
- 수정: 역방향 탐색 폐기 → A열 비어있고 K열 값 있는 합계행을 순방향으로 찾아 K·V·X 직접 읽기
- 추가 수정: `ss` 파라미터 대신 `sys` 변수 사용 (undefined 오류 방지)
